<h4 class="text-xl font-bold mb-2">Unit 2: Employee Leave System</h4>
<p class="mb-4">This Python program retains employee details and
allows them to book annual leave using a CLI interface. It explores
encapsulation and use of protected vs unprotected variables.</p>
<pre class="bg-gray-100 dark:bg-gray-700 p-4 rounded overflow-x-auto
text-sm">
<code>class Employee:
def __init__(self, name, leave_days):
self.name = name
self._leave_days = leave_days # protected variable
def request_leave(self, days):
if days <= self._leave_days:
self._leave_days -= days
return f"Leave approved. {self._leave_days} days remaining."
return "Leave request denied."
emp = Employee("John", 20)
print(emp.request_leave(5))</code>
</pre>
</div>
<!-- Unit 5 Artefact -->
<div class="bg-white dark:bg-gray-800 p-6 rounded-lg shadow mb-6">
<h4 class="text-xl font-bold mb-2">Unit 5: Polymorphism and Abstract
Classes</h4>
<p class="mb-4">This task involved using abstract base classes and
method overriding to create a flexible simulation of multiple robot
types.</p>
<pre class="bg-gray-100 dark:bg-gray-700 p-4 rounded overflow-x-auto
text-sm">
<code>from abc import ABC, abstractmethod
class Robot(ABC):
@abstractmethod
def perform_task(self):
pass
class DeliveryRobot(Robot):
def perform_task(self):
return "Delivering package..."
class CleaningRobot(Robot):
def perform_task(self):
return "Cleaning floor..."
robots = [DeliveryRobot(), CleaningRobot()]
for r in robots:
print(r.perform_task())</code>
</pre>
</div>
<!-- Unit 7 Artefact -->
<div class="bg-white dark:bg-gray-800 p-6 rounded-lg shadow mb-6">
<h4 class="text-xl font-bold mb-2">Unit 7: Data Structures and
Search</h4>
<p class="mb-4">A nested dictionary of cars is stored in a Car class,
and various dictionary methods are used to manipulate and display
data.</p>
<pre class="bg-gray-100 dark:bg-gray-700 p-4 rounded overflow-x-auto
text-sm">
<code>class Car:
def __init__(self):
self.data = {
"Toyota": {"year": 2020, "electric": False},
"Tesla": {"year": 2022, "electric": True}
}
def show_items(self):
return self.data.items()
def show_keys(self):
return self.data.keys()
def show_values(self):
return self.data.values()
car = Car()
print(list(car.show_items()))
print(list(car.show_keys()))
print(list(car.show_values()))</code>
</pre>

<!-- Assignment 2: Implementation -->
<div class="bg-white dark:bg-gray-800 p-6 rounded-lg shadow mb-6">
<h4 class="text-xl font-bold mb-2">Assignment: Implementation of the
Humanoid Robot System</h4>
<p class="mb-4">
Building on the design from Assignment 1, this Python
implementation included a command-line interface (`main.py`),
modular logic (`robot.py`), and unit tests (`test_robot.py`). The
robot class handled basic commands like start,
stop, move, and interact with users and its environment. It
simulated command stacking, battery level, and internal state.
</p>
<pre class="bg-gray-100 dark:bg-gray-700 p-4 rounded overflow-x-auto
text-sm mb-4">
<code>class HumanoidRobot:
def __init__(self, name):
self.name = name
self.active = False
self.battery = 100
def start(self):
if not self.active:
self.active = True
return f"{self.name} is now active."
return f"{self.name} is already active."
def charge(self):
self.battery = 100
return "Battery fully charged."
robot = HumanoidRobot("HAL-9000")
print(robot.start())
print(robot.charge())</code>
</pre>
<p>
<strong>Reflection:</strong> This final build tested everything I’d
learned in the module — from object-oriented structure
and CLI design to code documentation and unit testing. It
reinforced the importance of clean architecture and flexible code.
</p>
